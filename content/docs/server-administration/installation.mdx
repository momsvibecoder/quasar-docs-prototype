## 1.1. System Requirements

QuasarDB runs on several architectures, and requirements differ depending on which architecture you're targeting.

### 1.1.1. x64 build

- An Intel Haswell or better microarchitecture
- End-to-end ECC memory highly recommended
- At least 512 MiB of RAM
- At least 1 GiB of disk

### 1.1.2. Core2 build

The Core2 build is optimized for older CPUs, at a slight cost of performance.

<Callout type="note" title="Note">
  ARM64 builds may need special tuning to give you the best experience. Contact your Solution Architect for configuration recommendations.
</Callout>

## 1.2. Installation

The installation instruction may differ, depending on the operating system you are installing on.

<Tabs defaultValue="linux">
  <TabList>
    <Tab value="docker">Docker</Tab>
    <Tab value="windows">Windows</Tab>

  <TabPanel value="docker" title="Docker quick start">

```bash
docker run -p 2836-2837:2836-2837 -d --name qdb-server bureau14/qdb
docker run -ti --link qdb-server:qdb-server bureau14/qdbsh \
  --cluster qdb://qdb-server:2836
```

  </TabPanel>
  <TabPanel value="windows" title="Windows package installation">

```powershell
winget install QuasarDB.Server
qdbsh --cluster qdb://127.0.0.1:2836
```

  </TabPanel>
  <TabPanel value="linux" title="Linux package installation">

```bash
echo "deb [trusted=yes] https://repo.quasar.ai/stable/ /" \
  | sudo tee /etc/apt/sources.list.d/quasardb.list

apt install qdb-server qdbsh
systemctl enable --now quasardb
```

### Available images

| Image | Description | Tags | Core2 |
| --- | --- | --- | --- |
| `bureau14/qdb` | QuasarDB daemon | `latest`, `3.15.7` | yes |
| `bureau14/qdbsh` | QuasarDB shell | `latest`, `3.15.7` | yes |
| `bureau14/qdb-api` | API package image | `latest`, `3.15.7` | no |

### Environment variables

The QuasarDB docker containers provide several environment variables you can use to configure common settings.

| Variable | Usage | Default |
| --- | --- | --- |
| `QDB_CLUSTER` | Cluster URI | `qdb://127.0.0.1:2836` |
| `QDB_SECURITY_FILE` | User security file | empty |
| `QDB_LOG_LEVEL` | Log verbosity | `info` |
